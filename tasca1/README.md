# 🧑‍💻 Introducció a Git, GitHub i Control de Versions

## 👀 Primer contacte amb GitHub

Des de la vostra incorporació a **EverPia**, heu començat a sentir parlar d’eines i conceptes nous com:

- GitHub
- repositoris
- commits
- control de versions

Al principi poden semblar paraules estranyes, però formen part del dia a dia de qualsevol professional del sector IT.

---

# 🎯 Objectiu de la formació

Aquesta formació té com a objectiu:

- entendre què és el control de versions,
- conèixer el funcionament de **Git**,
- aprendre a utilitzar **GitHub**,
- i fer les primeres passes en documentació tècnica amb **Markdown**.

---

# 🧠 Què és el control de versions?

El control de versions ens permet:

- guardar l’històric de canvis,
- recuperar versions anteriors,
- treballar en equip sense conflictes,
- i mantenir el codi o la documentació organitzada.

---

# 🔄 Funcionament de Git

Git treballa amb tres estats principals:

## 📁 Working Directory
És on edites els fitxers.

## 📦 Staging Area
Zona intermèdia on prepares els canvis.

## 🗄️ Repository
On es guarden els canvis de forma definitiva (commits).

---

# 🧩 Conceptes clau

## 💾 Commit
És una “foto” dels canvis que has fet.

## 🌿 Branch
Una línia de treball paral·lela.

## ☁️ GitHub
Plataforma online per guardar i compartir repositoris Git.

---

# ✍️ Markdown

Markdown és un llenguatge de marcat que permet:

- escriure documentació de forma senzilla,
- formatar text (títols, llistes, enllaços),
- i crear documents llegibles i estructurats.

És la base de la documentació a GitHub.
